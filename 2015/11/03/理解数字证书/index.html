mod_wsgi, WSGI和uWSGI

发表于 2015-11-03

WSGI

Web服务器网关接口（Python Web Server Gateway Interface，缩写为WSGI）是为Python语言定义的Web服务器和Web应用程序或框架之间的一种简单而通用的接口。自从WSGI被开发出来以后，许多其它语言中也出现了类似接口。（来源：维基）

WSGI存在目的

在WSGI出现之前，对于Python一般而言，Web应用框架的选择将限制可用的Web服务器的选择，反之亦然。那时的Python应用程序（框架）通常是为CGI，FastCGI，mod_python中的一个而设计，甚至是为特定Web服务器的自定义的API接口而设计的。

WSGI （有时发音作’wiz-gee’）是作为Web服务器与Web应用程序（应用框架）之间的一种低级别的接口，以提升可移植Web应用开发的共同点。WSGI是基于现存的CGI标准而设计的。
所以，直接原因是为了移植性。

WSGI的特征

  • WSGI不是服务器，不是python模块，不是框架，不是API或是任何一种software. 它只是为了服务器和客户端交流，而设计一组接口规范。
  • 既然是规范，就要服务器和应用程序（框架）就都需要实现这个规范（规范细节详见PEP333）。
  • WSGI为了抹平应用程序（框架）和Web服务器之间的适用性差异，所以是位于服务器客户端的中间（中间件）
  • 对于客户端，WSGI中间件看上去是服务器;对于服务器，WSGI中间件看上去是客户端;

WSGI的实现

应用程序/框架端

创建一个”application”对象，callable，可以接受两个参数。

  • 一个类似CGI环境变量的字典对象
  • 一个将被“application”用于发送HTTP状态码/状态信息，HTTP头信息的方法

服务器端

具体不同Web服务器的实现是不同的，但都要满足以下要求。

  • 前提：服务器的基本功能是接受客户端请求
  • 实现了WSGI的服务器会把接收到的请求发给“应用/框架”实现的“callable application” object
  • 不仅是这样，它还会将请求客户端的信息转化成一个环境变量发给“application”（第一个参数）
  • 而且，服务器还会提供一个用于返回HTTP状态码，头信息的回调函数，传给“applicaiton”（第二个参数）
