Solr中的Filter, Analyzer, Tokenizer

发表于 2015-11-02

Preface

The following sections describe how Solr breaks down and works with textual data. There are three main concepts to understand:

  • Analyzers,
  • Tokenizers,
  • Filters.
    Field analyzers 的使用情景包括两个：建立索引时的截取过程和检索（query）过程.
    一个Analyzer检查一个Field中的文字，并且生成一组分割后的词组stream。Analyzers可能是一个单独的Class或者可以包括一系列的Tokenizer和Filter class。
    Tokenizers负责把字段中的字符串分割成一个个有语义的集合，或者Token。
    Filters 检查Tokenizer生成的Token流并且保存、转化、或者丢弃它们，或者创建一个新的Token（？）。Tokenizers和filters也许会合并成管道形式，或者合并成链（chains）形式，因此Tokenizer和Filter中某一个的输出可以作为下一个Tokenizer或Filter的输入。这样一个由tokenizer和filter组成的序列被称为一个analyzer。analyzer的输出结果通常被作为符合query条件的结果或者build indices（？）

Using Analyzers, Tokenizers, and Filters

虽然上述分析过程在建立索引和检索过程均有使用，但是这两种操作使用的分析过程却并不是同一个。

  • 对于建立索引过程，一般需要简化或者一般化单词/词组。比如，所有字母小写，消除标点和音调，使单词回到基本状态（mapping words to their stems）。这样做可以增加搜索引擎的‘回忆’（recall），比如 “ram”, “Ram” and “RAM” 都会检索到 “ram”.为了增加检索过程的精准程度，一个filter可以通过忽视all-cap acronyms（缩写）来实现缩小匹配范围的功能， if you’re interested in male sheep, but not Random Access Memory.

通过分析过程生成的Token定义了某个field的值，或者terms（术语），当增加一个文档时，该field不是用于建立那些terms的索引，就是去识别新增文档的哪部门包含你正在检索的terms。

简介

那麼我們來瞭解一下 Linux 使用的是哪一個 shell 呢？難道說 shell 不就是『一個 shell 嗎？』

例如常聽到的 Bourne SHell (sh) 、在 Sun 裡頭預設的 C SHell、 商業上常用的 K SHell、, 還有 TCSH 等等，每一種 Shell 都各有其特點。至於 Linux 使用的這一種版本就稱為 『 Bourne Again SHell (簡稱 bash) 』， 這個 Shell 是 Bourne Shell 的增強版本，也是基準於 GNU 的架構下發展出來的呦！

<注：本文全文使用Markdown语法>

概述

宗旨

Markdown 的目标是实现「易读易写」。

可读性，无论如何，都是最重要的。一份使用 Markdown 格式撰写的文件应该可以直接以纯文本发布，并且看起来不会像是由许多标签或是格式指令所构成。Markdown 语法受到一些既有 text-to-HTML 格式的影响，包括 Setext、atx、Textile、reStructuredText、Grutatext 和 EtText，而最大灵感来源其实是纯文本电子邮件的格式。

总之， Markdown 的语法全由一些符号所组成，这些符号经过精挑细选，其作用一目了然。

兼容 HTML

Markdown 语法的目标是：成为一种适用于网络的书写语言。
